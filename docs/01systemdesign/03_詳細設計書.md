## 1. 認証設計 (Auth.js v5)

### 1.1 ユーザー登録 - Server Action: register()

**実装:** `actions/auth.ts`

**入力（Zodスキーマ）:**
```typescript
const registerSchema = z.object({
  email: z.string().email(),
  password: z.string().min(6),
  name: z.string().min(1),
});
```

**処理:**
1. Zodバリデーション
2. BCryptハッシュ化
3. usersテーブルに挿入（role=CUSTOMER）
4. signIn()でJWT発行 + Cookie設定
5. redirect('/')

**エラー:**
| 条件 | 処理 |
|------|------|
| バリデーションエラー | Zodエラーメッセージを返却 |
| メールアドレス重複 | "このメールアドレスは既に登録されています" |

---

### 1.2 ログイン - Auth.js Credentials Provider

**実装:** `lib/auth.ts`

**入力:**
```typescript
const loginSchema = z.object({
  email: z.string().email(),
  password: z.string(),
});
```

**処理:**
1. メールアドレスでユーザー検索
2. BCrypt.verify(password, hash)
3. JWT発行 (claims: id, email, name, role)
4. HttpOnly Cookieに設定

**JWT Claims:**
| クレーム | 説明 |
|---------|------|
| id | ユーザーID (UUID) |
| email | メールアドレス |
| name | 表示名 |
| role | ロール名 (CUSTOMER/CREATOR/ADMIN) |

## 3. 調合機能

### 3.1 調合リクエスト作成 - Server Action: createBlend()

**実装:** `actions/blend.ts`

**入力（Zodスキーマ）:**
```typescript
const blendRequestSchema = z.object({
  topRatio: z.number().min(0).max(100),
  middleRatio: z.number().min(0).max(100),
  lastRatio: z.number().min(0).max(100),
  keywords: z.array(z.object({
    word: z.string().min(1),
    weight: z.number().min(0).max(10),
  })).min(1),
}).refine(d => d.topRatio + d.middleRatio + d.lastRatio === 100, {
  message: '比率の合計は100%にしてください',
});
```

**処理:**
1. 認証チェック（session取得）
2. Zodバリデーション
3. blend_requests + blend_request_keywords 保存
4. 調合計算（blend-calculator.ts）
5. blend_results + blend_result_flavors 保存
6. AIストーリー生成（非同期）
7. redirect(`/result/${id}`)

---

## 5. サービス層設計

### 5.1 認証 (Auth.js v5)

Auth.js Credentials Provider + JWT strategy. HttpOnly Cookie管理。

### 5.2 調合計算 (blend-calculator.ts)

**主要関数:**
| 関数 | 説明 |
|------|------|
| `generateBlend(request)` | メインの調合計算処理 |
| `calculateAutoBlend(keywords, rules, ratios)` | ルールベースの自動配合計算 |
| `normalize(weights: Map<string, Decimal>)` | 重みマップを正規化（合計=1.0） |

**自動配合アルゴリズム（Decimal.js使用）:**
```
noteFlavorWeights[noteType][flavorId] = Σ(rule.weight × keyword.weight)

normalizedWeights[noteType] = normalize(noteFlavorWeights[noteType])

finalRatio = normalizedWeights[flavorId] × (noteRatio / 100) × 100
```

### 5.3 ストーリー生成 (ai-client.ts)

| `generateStory(keywords, flavors, ratios)` | AIまたはフォールバックでストーリー生成 |
| `generateFallbackStory(keywords, flavors)` | テンプレートベースの代替ストーリー |

**Phase 1:** Vercel AI SDK (`ai` + `@ai-sdk/openai`)

## 6. フロントエンド コンポーネント設計

### 6.1 KeywordInput

| Props | 型 | 説明 |
|-------|-----|------|
| presets | Keyword[] | プリセットキーワード一覧 |
| selected | KeywordWeight[] | 選択中のキーワード+重み |
| onChange | (kw: KeywordWeight[]) => void | 変更コールバック |

**動作:**
- プリセットをカテゴリ別ボタンで表示（SEASON/PLACE/MOOD/FREE）
- フリーテキスト入力フィールド + 追加ボタン
- 選択済みキーワードにスライダー(0-10)で重みを設定
- 重複追加を防止

### 6.2 NoteRatioSlider

| Props | 型 | 説明 |
|-------|-----|------|
| topRatio / middleRatio / lastRatio | number | 現在の比率 |
| onTopChange / onMiddleChange / onLastChange | (v: number) => void | 変更コールバック |

**動作:**
- TOP: 30〜40%、MIDDLE: 50〜60%、LAST: 10〜15% の制約付きスライダー
- 合計100%のバリデーション表示
- カラーバーで視覚的に比率を表示

### 6.3 FlavorChart

| Props | 型 | 説明 |
|-------|-----|------|
| result | BlendResult | 配合結果 |

**動作:** Recharts PieChart で全香料の配合比率を円グラフ表示。ノートタイプ別に色分け。

### 6.4 StoryDisplay

| Props | 型 | 説明 |
|-------|-----|------|
| story | string | AIストーリー |

**動作:** モノトーンのカード内にストーリーテキストを表示。

## 9. JWT トークン仕様

| 項目 | 値 |
|------|-----|
| アルゴリズム | HS256 |
| シークレット | 環境変数 AUTH_SECRET（256bit以上） |
| 有効期限 | 24時間（86400000ms） |

**Claims:**
| クレーム | 説明 |
|---------|------|
| sub | ユーザーID (UUID) |
| email | メールアドレス |
| name | 表示名 |
| role | ロール名 (CUSTOMER/CREATOR/ADMIN) |
| iat | 発行時刻 |
| exp | 有効期限 |

**Cookie設定:**
- Name: `authjs.session-token`
- HttpOnly: true
- Secure: true (本番)
- SameSite: Lax
