# テーマ軸レイヤーアーキテクチャ

## 概要

ユーザーの香り選択からAI配合・ストーリー生成までを5層構造で設計する。
上位レイヤーほど配合計算に直結し、下位レイヤーほどストーリー・演出に寄与する。

---

## レイヤー構成

```
┌─────────────────────────────────────────────────────┐
│  L1  物理層        香りの物理的性質を定義             │  ← 配合計算に直結
│      (Physical)    調香師が香料ごとに入力             │
├─────────────────────────────────────────────────────┤
│  L2  環境層        いつ・どこで使うかの制約条件        │  ← レシピ選択の絞り込み
│      (Context)     ユーザーが選択                    │
├─────────────────────────────────────────────────────┤
│  L3  印象層        どんな印象を与えたいか             │  ← 香料の組み合わせ方針
│      (Impression)  ユーザーが選択                    │
├─────────────────────────────────────────────────────┤
│  L4  世界観層      ビジュアル・物語の方向性            │  ← ストーリー生成・UI演出
│      (Narrative)   ユーザーが選択 or AIが推論         │
├─────────────────────────────────────────────────────┤
│  L5  関係層        誰のため・どんな関係で             │  ← パーソナライズ・EC連携
│      (Relation)    ユーザーが選択（任意）             │
└─────────────────────────────────────────────────────┘
```

---

## L1: 物理層（Physical）

香料そのものの物理特性。**配合比率の計算に直接影響**する。
調香師が香料マスタ登録時に入力。

| ID | 軸 | 項目例 | 型 | 用途 |
|----|---|-------|---|------|
| 1-1 | ノートタイプ | TOP / MIDDLE / LAST | enum | ノートバランス制約 |
| 1-2 | 揮発性 | 0.00〜1.00 | float | 持続時間の計算 |
| 1-3 | 香りの強さ | 0.00〜1.00 | float | 配合上限の計算 |
| 1-4 | カテゴリ | citrus / floral_light / floral_rich / green / marine / wood / spice | enum | 香料グルーピング |
| 1-5 | 重さ | エアリー / バランス / リッチ | enum | 全体の重量感設計 |
| 1-6 | 温度感 | クール / ニュートラル / ウォーム | enum | フレグランスマップ座標 |

- 入力単位: 1香料ごと（香料マスタに統合）
- テーマ軸としては使わない（レシピ生成ロジックが参照する内部パラメータ）
- 1-5, 1-6 は `flavor-coordinates.ts` のx/y座標と対応

---

## L2: 環境層（Context）

香りを**使う状況**。ユーザーが「いつ・どこで」を選ぶ。

| ID | 軸 | 項目 | 数 | 入力者 | 必須度 |
|----|---|------|---|-------|--------|
| 2-1 | 季節 | 春 / 夏 / 秋 / 冬 | 4 | ユーザー選択 | 主軸 |
| 2-2 | シーン（TPO） | デート / オフィス / パーティー / リラックス / アウトドア / フォーマル | 6 | ユーザー選択 | 主軸 |
| 2-3 | 時間帯 | 朝 / 昼 / 夜 | 3 | ユーザー選択 | 補助 |
| 2-4 | 天候 | 晴れ / 雨 / 曇り / 雪 | 4 | AI推論 | 自動 |
| 2-5 | 場所 | 室内 / 屋外 / 移動中 | 3 | AI推論 | 自動 |

- 2-1 季節, 2-2 シーンが**レシピの主軸**
- 2-3 時間帯は補助軸（レシピにタグとして付与）
- 2-4, 2-5 は L2-1, L2-2 から AI 推論（調香師入力不要）

---

## L3: 印象層（Impression）

**どんな印象の香りか**。ユーザーの感覚的な好み。

| ID | 軸 | 項目 | 数 | 入力者 | 必須度 |
|----|---|------|---|-------|--------|
| 3-1 | ムード | 明るい / 落ち着いた / 情熱的 / ミステリアス / 清潔感 | 5 | ユーザー選択 | 主軸 |
| 3-2 | エネルギー | アクティブ / リラックス / 集中 / 高揚 | 4 | ユーザー選択 | 補助 |
| 3-3 | 香りの強さ | ライト / ミディアム / インテンス | 3 | ユーザー選択 | 補助 |
| 3-4 | 持続イメージ | 一瞬の印象 / 半日 / 一日中 | 3 | ユーザー選択 | 補助 |
| 3-5 | 質感 | シルキー / クリスプ / パウダリー / ジューシー | 4 | ユーザー選択 | 補助 |

- 3-1 ムードが**レシピの主軸**
- 3-2〜3-5 は補助（レシピにタグとして付与、またはAI推論）
- 3-3, 3-4 は L1 物理パラメータへの変換ブリッジ

## テーマ構造（レシピ管理）

### 主軸（テーマの軸）

| レイヤー | 軸 | 項目数 |
|---------|---|--------|
| L2 環境層 | シーン | 6 |
| L3 印象層 | ムード | 5 |

**基本テーマ数: 6 × 5 = 30テーマ**

### テーマとクリエーターレシピの関係

30テーマの下にクリエーターのシグネチャーレシピが紐づく構造:

```
30テーマ（プラットフォーム定義）
├── デート x 明るい
│   ├── ベースAIルール（perfumer_id=NULL）→ 常に利用可能
│   ├── クリエーターA: レシピ「春風のデート」
│   ├── クリエーターB: レシピ「シトラスデート」
│   └── ...
├── デート x 落ち着いた
│   ├── ベースAIルール（perfumer_id=NULL）→ 常に利用可能
│   ├── クリエーターA: レシピ「静かな午後」
│   └── ...
├── オフィス x 清潔感
│   ├── ベースAIルール（perfumer_id=NULL）→ 常に利用可能
│   └── ...
└── ... (30テーマ全て)
```

ベースAIルールは全30テーマを網羅するため、クリエーターレシピがなくても調合可能。

### 副軸（レシピにタグとして付与）

| レイヤー | 軸 | 項目数 | 付与方式 |
|---------|---|--------|---------|
| L2 環境層 | 季節 | 4 | 複数選択可（春夏秋冬） |
| L2 環境層 | 時間帯 | 3 | 複数選択可（朝昼夜） |
| L3 印象層 | エネルギー | 4 | 単一選択 |
| L3 印象層 | 香りの強さ | 3 | 単一選択 |

### 目標レシピ数

- ベースAI: 30テーマ分のルールをプラットフォーム管理者が定義（必須）
- クリエーターレシピ: 各クリエーターが自由にテーマを選んでレシピを作成
- プラットフォーム全体目標: 30テーマ × 各10レシピ以上 = **300件以上**

## レイヤー間の依存関係

```
L1 物理層 ←──── 調香師が香料ごとに定義（マスタデータ）
  ↑ 参照
L2 環境層 ←──── ユーザー選択 → テーマ絞り込み（シーン）
  ↓ 制約           ↓ 推論
L3 印象層 ←──── ユーザー選択 → テーマ絞り込み（ムード）
  ↓ 入力           ↓
  ↓              クリエーター/レシピ層（ハイブリッド型）
  ↓                ├── ベースAIルール（perfumer_id=NULL）→ 全テーマ対応
  ↓                ├── クリエータースタイル → ベースAI配合の補正
  ↓                └── シグネチャーレシピ → テーマに紐付く完成品配合
  ↓ 入力           ↓ 推論
L4 世界観層 ←─── 大部分をAIが L2+L3 から自動推論
  ↓ 補足           ↓ 推論
L5 関係層 ←──── ユーザー任意選択 → パーソナライズ・EC
```

## 各レイヤーの役割まとめ

| レイヤー | 調香師入力 | ユーザー選択 | AI推論 | 影響範囲 | レシピとの関連 |
|---------|:---------:|:----------:|:------:|---------|-------------|
| L1 物理層 | 必須 | - | - | 配合計算 | レシピ配合の基礎データ |
| L2 環境層 | レシピ主軸+タグ | 主要2軸 | 補助軸 | レシピ絞り込み | レシピの`scene`カラムに対応 |
| L3 印象層 | レシピ主軸+タグ | 主要1軸 | 補助軸 | 組み合わせ方針 | レシピの`mood`カラムに対応 |
| L4 世界観層 | コンセプト文 | 任意1-2軸 | 大半 | ストーリー・UI | レシピの`concept`から間接反映 |
| L5 関係層 | - | 任意 | 一部 | EC・パーソナライズ | レシピ→商品のEC連携 |
