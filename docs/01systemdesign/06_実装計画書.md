```typescript
creatorProfiles:

  - displayName: VARCHAR(100)
```

## 5. Sprint 2: Phase 2 — クリエーター機能（Week 2）

### 5.1 クリエーターサービス層

**新規ファイル:** `lib/services/creator-profile-service.ts`

```
- getActiveCreators(): クリエーター一覧取得（公開・承認済みのみ）

- toggleFavorite(userId, creatorId): お気に入り登録/解除
- getPendingCreators(): 承認待ち一覧（管理者用）
- approveCreator(id): 承認（管理者用）
- rejectCreator(id): 却下（管理者用）
```

### 5.2 調合計算のクリエーター対応

**変更ファイル:** `lib/services/blend-service.ts`

```
- executeBlend() に creatorId パラメータ追加
- ルール取得時に creatorId でフィルタリング
  - creatorId 指定あり → そのクリエーターのルールのみ
  - creatorId 指定なし → 全ルール（従来動作、後方互換）
- BlendRequest レコードに creatorId を保存
```

**変更ファイル:** `lib/ai-client.ts`

```
- generateStory() に creatorのstyleDescription を追加パラメータ
- suggestRules() に creatorのstylePrompt を追加パラメータ
```

### 5.3 Server Actions

**新規ファイル:** `actions/creators.ts`

```
- createBlendWithCreator(creatorId, ...): クリエーター指定調合
```

**新規ファイル:** `actions/admin.ts`（既存を拡張）

## 8. Sprint 5: Phase 1.5 — Python AI Service

### 8.1 概要

現在 `ai-client.ts` で直接OpenAIを呼び出しているAI処理を、Python FastAPIサービスに分離する。

### 8.2 構成

```
apps/ai-service/
├── Dockerfile
├── pyproject.toml
├── app/
│   ├── main.py              # FastAPIアプリ
│   ├── config.py            # pydantic-settings
│   ├── routers/
│   │   ├── story.py         # POST /ai/story/generate
│   │   ├── rules.py         # POST /ai/rules/suggest
│   │   │                    # POST /ai/rules/suggest-bulk
│   │   └── health.py        # GET /ai/health
│   ├── services/
│   │   ├── story_service.py
│   │   ├── rule_suggestion_service.py
│   │   └── openai_client.py
│   ├── prompts/              # Jinja2テンプレート
│   │   ├── story_system.j2
│   │   ├── story_user.j2
│   │   ├── rule_suggest_system.j2
│   │   └── rule_suggest_user.j2
│   └── models/               # Pydanticモデル
└── tests/
```

### 8.3 エンドポイント

| メソッド | パス | 用途 | Phase |
|---------|------|------|-------|
| POST | `/ai/story/generate` | ストーリー生成（同期） | 1.5 |
| POST | `/ai/rules/suggest` | ルール提案（同期） | 1.5 |
| POST | `/ai/rules/suggest-bulk` | 一括ルール提案 | 1.5 |
| GET | `/ai/health` | ヘルスチェック | 1.5 |
| POST | `/ai/preference/analyze` | 嗜好分析 | 4 |
| POST | `/ai/recommend/creators` | クリエーター推薦 | 4 |

### 8.4 移行手順

```
1. FastAPIプロジェクト作成、Dockerfileビルド確認
2. story_service.py にストーリー生成ロジック実装
3. rule_suggestion_service.py にルール提案ロジック実装
4. フォールバックロジック移植
5. Next.js の ai-client.ts を Python呼出しに差替え
   （インターフェースは変えず、内部のHTTP先を変更するのみ）
6. docker-compose.yml の ai-service profile: full → デフォルトに変更
7. 既存テストが全て通ることを確認
```

### 8.5 認証

```
- Next.js → Python: INTERNAL_SERVICE_TOKEN による Bearer認証
- ユーザートークンは不要（内部サービス間通信）
```
