### 3.3 認証フロー

```
[ユーザー] → Credentials Provider (Auth.js v5)
                ↓
[Auth.js] → BCrypt検証 → JWT発行（role含む）
                ↓
[レスポンス] ← HttpOnly Cookie に JWT を設定
                ↓
[以降のリクエスト] → Cookie 自動送信
                ↓
[middleware.ts] → JWT検証 → ロール別ルーティング
[Server Actions] → session + role 検証
```

### 3.4 調合生成フロー

#### 自動生成モード（キーワードベース）

```
1. ユーザーがキーワード + ノート比率を送信
2. 各キーワードのルール(keyword_flavor_rules)を取得
3. ルール重み × ユーザー重み で統合スコアを計算
4. ノートタイプ別に正規化（合計=1.0）
5. ノート比率を乗算して最終配合比率を算出
6. Vercel AI SDK（Phase 1）/ Python AI Service（Phase 1.5）でストーリーを生成
7. 結果を保存して返却
```

**計算例:**
```
キーワード「夏」(weight=7)
  ルール: ライム(TOP) weight=0.8 → 0.8 × 7 = 5.6
  ルール: ベルガモット(TOP) weight=0.6 → 0.6 × 7 = 4.2

TOPノート正規化: ライム = 5.6/(5.6+4.2) = 0.571
                ベルガモット = 4.2/(5.6+4.2) = 0.429

TOPノート比率 = 35% の場合:
  ライム = 0.571 × 0.35 × 100 = 20.00%
  ベルガモット = 0.429 × 0.35 × 100 = 15.00%
```

### 3.5 AIルール提案フロー

```
1. 調香師がキーワードの「AI提案」をクリック
2. POST /api/perfumer/keywords/{id}/suggest
3. RuleSuggestionService:
   a. 全香料リストを取得
   b. プロンプトを構築（キーワード + 香料一覧 + JSON形式指定）
   c. OpenAI API呼び出し (temperature=0.5)
   d. JSONレスポンスをパース
   e. 香料名(英語)でFlavorエンティティとマッチング
   f. 重みを0.05単位に丸め
4. 提案ルールをフロントに返却
5. 調香師がレビュー・調整後に保存
```

### 6.2 調合生成ロジック変更（フェーズ2）

```
■ 現行（フェーズ1）
rules = findRulesByKeyword(keyword)
→ 全クリエーターのルールを統合

■ フェーズ2
if request.creatorId exists:
    rules = findRulesByKeywordAndCreator(keyword, creatorId)
    → 選択クリエーターのルールのみ使用
else:
    rules = findRulesByKeyword(keyword)
    → フォールバック: 全ルール統合
```

> **既存DB対応**: `keyword_flavor_rules.perfumer_id` カラムは V1 で定義済み。
> クエリに `findRulesByKeywordAndCreator(keyword, creatorId)` を追加するだけで対応可能。
