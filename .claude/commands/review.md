# /review

現在の変更差分をコードレビューしてください。

## レビュー対象

1. `git diff` でステージング済み + 未ステージの変更を取得
2. 変更がない場合は直近のコミットをレビュー

## レビュー観点

以下の観点でレビューし、問題があれば指摘:

### 必須チェック
- **型安全性**: any の使用、型アサーションの濫用
- **セキュリティ**: SQLインジェクション、XSS、認証漏れ、機密情報の露出
- **エラーハンドリング**: try-catch の不足、エラーメッセージの適切さ
- **バリデーション**: Zod スキーマの不足、入力値チェック

### 推奨チェック
- **パフォーマンス**: 不要な再レンダリング、N+1クエリ
- **コード品質**: 重複コード、命名の適切さ、関数の長さ
- **設計書との整合性**: docs/ の仕様との乖離

## 出力形式

```
コードレビュー結果:

🔴 Critical (修正必須):
  - file.ts:42 — SQLインジェクションの可能性

🟡 Warning (推奨):
  - page.tsx:15 — Server Component で十分な処理にuse clientが付いている

🟢 Good:
  - トランザクション処理が適切に実装されている
```
