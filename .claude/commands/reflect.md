# セッション振り返り

このセッションでの学びを Auto Memory に記録する。

## 手順

1. 今回のセッションで行った作業を振り返る
2. 以下の観点で学びを抽出する:
   - **修正パターン**: ユーザーから修正された点（corrections.md に追記）
   - **意思決定**: 技術選定やアーキテクチャ判断（decisions.md に追記）
   - **好みの発見**: ユーザーの新しい好みが確認された場合（preferences.md に追記）
   - **プロジェクト構造**: 重要なファイルパスやパターンの変更（MEMORY.md を更新）
3. 既存のメモリファイルを読み、重複がないか確認する
4. 該当するファイルに追記・更新する
5. 記録した内容のサマリを表示する

## メモリファイルの場所

- `~/.claude/projects/-Users-yoshioka-work-aisent2026/memory/MEMORY.md`
- `~/.claude/projects/-Users-yoshioka-work-aisent2026/memory/corrections.md`
- `~/.claude/projects/-Users-yoshioka-work-aisent2026/memory/decisions.md`
- `~/.claude/projects/-Users-yoshioka-work-aisent2026/memory/preferences.md`

## ルール

- 推測ではなく、実際に確認された事実のみ記録する
- 既存の記録と矛盾する場合は、新しい情報で更新する
- セッション固有の一時的な情報は記録しない
- MEMORY.md は 200 行以内に収める
